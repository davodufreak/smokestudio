🔴 Incidencias Críticas

Bloqueantes de producción — requieren corrección inmediata

C-01

Firebase no configurado — Autenticación completamente rota

Crítico
smokescan.js · líneas 1135–1141

SmokeScan no puede autenticar usuarios ni persistir datos. OAuth Google/Microsoft completamente no funcional.

const firebaseConfig = {
  apiKey: "TU_API_KEY",
  authDomain: "tu-proyecto.firebaseapp.com",
  // ...
};

Crear proyecto Firebase real, configurar variables de entorno (no hardcodear), activar Google/Microsoft OAuth providers. Nunca commitar credenciales reales.

C-02

API Key de Gemini ausente — Función AI completamente rota

Crítico
smokescan.js · STATE.apiKey

La característica core del producto (análisis AI) no funciona. El sistema cae en modo simulado con respuestas hardcodeadas.

Implementar un backend proxy (Firebase Cloud Functions o Vercel Edge Functions) para las llamadas a Gemini. Nunca exponer API keys en el frontend.

C-03

Sin backend — API keys expuestas si se configuran en frontend

Crítico
smokescan.js · fetchGeminiAnalysis()

Sin servidor backend, cualquier API key real configurada en el código frontend quedará expuesta públicamente. Riesgo de abuso masivo y costos ilimitados.

Implementar backend serverless como proxy para todas las llamadas externas. Opciones: Firebase Cloud Functions, Vercel Serverless, o Cloudflare Workers.

🟠 Incidencias Altas

Problemas de seguridad y rendimiento significativos

A-01

Vulnerabilidad XSS — innerHTML con datos de usuario sin sanitizar

Alto
smokescan.js · ~línea 1250

Permite inyección de HTML/scripts maliciosos si un atacante controla el nombre de usuario. Patrón peligroso que debe eliminarse.

// ❌ Problemático
cta.innerHTML = 'Hola, ' + firstName;

// ✅ Correcto
cta.textContent = `Hola, ${firstName}`;
A-02

Control de intentos únicamente en cliente — Bypass trivial

Alto
smokescan.js · FirebaseAuthService

El límite de 2 intentos/día se resetea al recargar la página. Cualquier usuario puede usar SmokeScan ilimitado veces generando costos no controlados de API.

A-03

Autenticación simulada activa — Mock mode en codebase

Alto
smokescan.js · FirebaseAuthService

Los usuarios ven un modal de login falso. Si se despliega en este estado, el producto no funciona. console.log filtra información de debugging en producción.

A-04

Sin reglas de seguridad en Firestore

Alto
firestore.rules (ausente)

Sin reglas configuradas, Firestore en modo development permite leer/escribir cualquier dato a cualquier usuario. Exposición total de datos en producción.

A-05

Imágenes sin optimizar — Hasta 1.6 MB por imagen

Alto
problem-stats.jpg (1.6MB) · hero-team.jpg (1.1MB) · cta-team.jpg (945KB)

En conexiones móviles el LCP supera 4 segundos, penalizando SEO y conversión. Total de assets: 75 MB.

Comprimir a ≤200KB por imagen, generar variantes WebP, implementar srcset con breakpoints en 400/800/1200px, añadir loading="lazy" en imágenes fuera del viewport inicial.
